# Introdução

Para a ideação do projeto, foram analisados sistemas já existentes de inspiração para a UI. Sendo assim, foram escolhidos 4 divisões que cada uma tem sua inspiração:

# Inspiração de UI
![UI_examples](/img/UI_examples.png)

1- Menu vazado - Dispatch -> Foi escolhido por mostrar as informações necessárias, mas ao mesmo tempo mostrando a cena, sendo um menu vazado.

2- Escolha de cores - Amazon -> Foi escolhido por ser minimalista, o que faria sentido com o menu vazado, e também uma seleção de cores que o usuário já está familiarizado.

3- Barra de seleção horizontal - Clash of Clans/The Sims 4/Genshin impact -> Barra horizontal e contendo imagem do objeto e filtros em cima.

# Inspiração de poiscionamento de objetos
![placement_examplev](/img/placement_example.png)

1- Posicionamento - Palworld -> Visão de o que está sendo posicionado, em destaque para se consegue ser posicionado naquela região ou não.

2- Posicionamento - VR -> Visão melhor de como o posicionamento funcionaria no VR.

3- Grid - The Sims 4 -> Visão do grid no posicionamento.

# Inspiração de objetos posicionados 
![position_example](/img/position_example.png)

1- Visão - VR -> Como os objetos estarão sendo mostrados após serem colocados, em VR.

2- Visão - Forza -> Como os objetos se misturam ao ambiente, como o adesivo no carro.

# Inspiração de cores
![color_examples](/img/color_examples.png)

1- Cores para posicionamento - Palworld -> Destaque em azul ou em vermelho para visualização da possibilidade de colocar o objeto em devido lugar.

2- Grid - The Sims 4 -> Grid verde, o que permite uma visualização clara do grid e contraponto ao ambiente, para ajudar ao usuário colocar os objetos.

## Considerações de UI/UX

- **Familiaridade:** Utilizar padrões já conhecidos pelo usuário aumenta a velocidade de entendimento da interface.  
- **Minimalismo:** O menu vazado reforça a ideia de imersão e evita que elementos visuais ocupem espaço desnecessário.  
- **Feedback visual:** O uso de cores (ex.: verde/vermelho/azul) comunica estados sem necessidade de texto.  
- **Acessibilidade:** Elementos bem espaçados e ícones claros tornam a tela mais intuitiva.

# Conclusão

A tela de escolha de objeto foi desenvolvida como uma extensão natural do restante do sistema, combinando inspiração de jogos modernos com elementos de interfaces amplamente utilizadas no cotidiano. Ao utilizar um menu vazado, um mini tutorial contextual e seletores familiares, a experiência do usuário se torna clara, fluida e acessível — mesmo para aqueles que estão experimentando o sistema pela primeira vez.  
Essa abordagem garante não apenas consistência visual, mas também um fluxo de interação eficiente, reforçando o objetivo principal do projeto: proporcionar uma experiência intuitiva, moderna e agradável durante o processo de seleção e personalização de objetos.
